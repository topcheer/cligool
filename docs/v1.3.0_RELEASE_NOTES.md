# CliGool v1.3.0 - 无状态架构重构

## 🎉 重大架构升级

**CliGool** 是一个基于Go和WebSocket的**跨平台远程终端解决方案**，让您的本地终端可以通过任何Web浏览器访问。

## 🚀 v1.3.0 核心更新

### 🏗️ 无状态架构重构
- ✅ **移除数据库依赖** - 不再需要PostgreSQL和Redis
- ✅ **内存会话管理** - 会话状态在内存中维护，90秒无活动自动清理
- ✅ **简化部署** - 从3个服务简化为1个服务（relay only）
- ✅ **降低资源占用** - 减少依赖和运行时资源消耗
- ✅ **更易水平扩展** - 无状态设计，易于负载均衡

### 🐳 Docker部署优化
- ✅ **生产环境配置** - 使用预构建镜像快速部署
- ✅ **开发环境配置** - 本地构建和热重载支持
- ✅ **配置文件分离** - `docker-compose.yml`（生产）vs `docker-compose.dev.yml`（开发）

### ☁️ 云平台支持增强
- ✅ **Koyeb平台** - 修复健康检查端口配置（8080而非8000）
- ✅ **Koyeb配置** - 添加`koyeb.yaml`和优化的`Dockerfile.koyeb`
- ✅ **Railway平台** - 配置文件优化
- ✅ **Fly.io平台** - 配置文件优化
- ✅ **Render平台** - Blueprint配置文件

### 🐛 Bug修复
- ✅ **URL拼接修复** - 修复server参数带末尾斜杠时生成的双斜杠问题
- ✅ **健康检查修复** - Koyeb平台TCP健康检查端口错误

### 📚 文档全面更新
- ✅ 移除所有数据库和Redis相关说明
- ✅ 更新部署指南，反映无状态架构
- ✅ 更新Docker使用文档
- ✅ 更新开发指南
- ✅ 更新项目总结文档

## 🚀 快速开始

### 方式1：Docker Compose（推荐）

```bash
# 克隆项目
git clone https://github.com/topcheer/cligool.git
cd cligool

# 生产环境（使用预构建镜像）
docker-compose up -d

# 或：开发环境（本地构建）
docker-compose -f docker-compose.dev.yml up -d --build

# 访问服务
open http://localhost:8081
```

### 方式2：云平台一键部署

**Railway**:
1. 访问 [railway.app](https://railway.app)
2. 点击 "Deploy from GitHub"
3. 选择 `topcheer/cligool` 仓库
4. Railway会自动检测`railway.toml`配置并部署

**Render**:
1. 访问 [render.com](https://render.com)
2. 点击 "New Blueprint Instance"
3. 连接GitHub仓库
4. Render会自动读取`render.yaml`配置

**Fly.io**:
```bash
flyctl launch
```
会自动读取`fly.toml`配置。

**Koyeb**:
```bash
koyeb init
```
会自动读取`koyeb.yaml`配置。

### 方式3：本地运行客户端

```bash
# 下载客户端
curl -sSL https://raw.githubusercontent.com/topcheer/cligool/main/install.sh | bash

# 连接到服务器
cligool -server https://your-server.com

# 或使用配置文件
cligool  # 自动使用 ~/.cligool.json 配置
```

## 🔧 配置变更

### 环境变量（简化）
| 变量 | 说明 | 默认值 |
|------|------|--------|
| `RELAY_HOST` | 服务监听地址 | 0.0.0.0 |
| `RELAY_PORT` | 服务监听端口 | 8080 |

### 移除的环境变量
- ❌ `DATABASE_URL` - 不再需要
- ❌ `REDIS_URL` - 不再需要
- ❌ `ENABLE_AUTO_HTTPS` - 使用反向代理代替

### 客户端配置文件 (~/.cligool.json)
```json
{
  "server": "https://your-server.com",
  "cols": 120,
  "rows": 36
}
```

## 📊 架构对比

### v1.2.0 及之前（有状态）
```
Web客户端 ──► 中继服务器 ──► PostgreSQL
                         └──► Redis
                         └──► CLI客户端
```

### v1.3.0（无状态）✨
```
Web客户端 ──► 中继服务器 ──► CLI客户端
                   │
                   └──► 内存会话管理
```

**优势**：
- 部署更简单（1个服务 vs 3个服务）
- 资源占用更少
- 更容易水平扩展
- 更快的启动速度

## ⚡ 核心功能

- **低延迟** - WebSocket毫秒级响应
- **真实PTY** - 完整的终端特性支持
- **AI CLI工具** - 完美支持Claude、Gemini、Aider等
- **多用户协作** - 多人可同时连接同一终端会话
- **自动会话清理** - 90秒无活动自动断开
- **心跳保活** - 30秒间隔，90秒超时
- **跨平台支持** - 30个操作系统和架构

## 🔄 升级指南

### 从 v1.2.0 升级到 v1.3.0

**Docker部署**：
```bash
# 停止旧版本
docker-compose down

# 拉取新代码
git pull origin main

# 启动新版本（不再需要启动postgres和redis）
docker-compose up -d

# 删除旧的数据库容器（可选）
docker rm -f cligool-postgres cligool-redis
docker volume rm cligool_postgres_data cligool_redis_data
```

**云平台部署**：
- 直接部署最新代码即可
- 所有云平台配置已更新为无状态架构

**客户端**：
- 无需特殊升级，直接下载新版本即可
- 配置文件格式保持兼容
