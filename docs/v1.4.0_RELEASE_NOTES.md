# CliGool v1.4.0 - Web 终端用户体验全面升级

## 🎉 重大用户体验改进

**CliGool** 是一个基于Go和WebSocket的**跨平台远程终端解决方案**，让您的本地终端可以通过任何Web浏览器访问。

## 🚀 v1.4.0 核心更新

### 💾 消息缓存机制
- ✅ **自动缓存CLI消息** - 当没有Web客户端连接时，服务器自动缓存CLI输出（最多1000条消息）
- ✅ **历史恢复** - Web客户端连接时自动接收并显示所有缓存的历史消息
- ✅ **无缝体验** - 用户可以随时打开浏览器查看完整的终端历史，不会错过任何输出
- ✅ **智能限制** - 缓存大小限制（1000条/10MB），防止内存溢出

**使用场景**：
- CLI客户端先启动，Web客户端后连接
- CLI执行命令时没有Web客户端查看
- 多个Web客户端轮流连接同一session

### 🎨 平滑终端初始化
- ✅ **消除视觉闪烁** - 移除临时欢迎信息，终端保持空白直到收到真实内容
- ✅ **智能清理** - 收到第一个WebSocket消息时清理终端，然后平滑渲染
- ✅ **专业体验** - 像现代Web应用一样流畅的加载体验

### 🖥️ 自动隐藏UI
- ✅ **全屏终端** - WebSocket连接成功后自动隐藏页头和页尾
- ✅ **沉浸式体验** - 终端占满整个屏幕，类似本地终端的感觉
- ✅ **智能显示** - 断开连接后自动恢复UI，方便查看连接状态

### 🤖 无CLI客户端友好提示
- ✅ **清晰错误说明** - Web客户端连接到不存在的session时显示友好提示
- ✅ **平台特定命令** - 自动生成适合当前操作系统的启动命令示例
- ✅ **美观界面** - 使用ANSI颜色和框线绘制的专业终端界面
- ✅ **下载链接** - 包含客户端下载地址和使用提示

### 🔔 CLI断开通知
- ✅ **实时通知** - CLI客户端断开时立即通知所有Web客户端
- ✅ **友好提示** - 显示清晰的断开消息和原因
- ✅ **重连指导** - 提示用户确保CLI客户端正在运行

### 🛠️ 错误处理改进
- ✅ **完善的清理** - 使用defer确保WebSocket连接总是被正确清理
- ✅ **错误消息** - 命令执行失败时发送详细的错误关闭消息
- ✅ **避免死锁** - 移除log.Fatalf，确保defer函数执行

### 🔇 移除不必要的消息
- ✅ **干净的终端** - 移除所有连接状态和欢迎消息
- ✅ **保留调试** - 保留浏览器控制台日志用于调试
- ✅ **专注输出** - 用户只看到真实的终端输出，无干扰

### ⚙️ -no-browser 参数
- ✅ **禁用自动打开** - 添加`-no-browser`参数禁止自动打开浏览器
- ✅ **配置文件支持** - 可在`~/.cligool.json`中设置`no_browser: true`
- ✅ **适用场景** - 服务器环境、SSH会话、自动化脚本

- ✅ 新增7个功能文档，详细说明每个改进
- ✅ 新增Web终端改进总结文档

## 🚀 快速开始

### Docker 部署（推荐）

```bash
# 克隆仓库
git clone https://github.com/zty8cn/cligool.git
cd cligool

# 启动服务
docker-compose up -d

# 访问 http://localhost:8081
```

### 本地运行

```bash
# 1. 启动中继服务器
go run ./cmd/relay

# 2. 启动客户端（另一个终端）
./bin/cligool-darwin-arm64

# 3. 打开浏览器访问显示的URL
```

### 使用新功能

```bash
# 禁止自动打开浏览器
./cligool -server http://localhost:8081 -no-browser

# 使用配置文件
cat > ~/.cligool.json << EOF
{
  "server": "http://localhost:8081",
  "no_browser": true
}
EOF

./cligool
```
