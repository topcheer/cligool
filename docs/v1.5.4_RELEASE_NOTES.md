# CliGool v1.5.4 - CLI Relay Auto Reconnect

## 🎯 本次更新重点

**CliGool v1.5.4** 重点增强 CLI 到 relay 的连接可靠性：当 relay 或代理链路暂时不可用时，CLI 不再直接失败退出，而是保持本地终端继续运行、自动重连，并在恢复后按顺序回放未发送消息。

## ✅ 更新内容

### 🔁 CLI 自动重连 relay / 代理链路
- CLI 客户端在本地 PTY / ConPTY 启动后，后台持续尝试连接 relay
- relay 或代理连接异常时，CLI 会进入自动重试，而不是直接退出
- 重连成功后会明确输出连接恢复提示

### 📦 未发送消息顺序缓冲与回放
- WebSocket 未发送成功的客户端消息会暂存在本地内存中
- 连接恢复后先重新发送 `init`
- 再按原始顺序回放缓冲的 `output`、`resize` 等消息

### 🖥️ 本地终端优先，远端恢复后自动同步
- 即使 relay 暂时不可达，本地 PTY / ConPTY 仍可继续运行
- Unix 与 Windows 客户端都统一接入新的连接管理逻辑
- 代理连接场景复用同一拨号路径，行为与直连保持一致

### 🧪 新增重连回归测试
- 增加连接建立前缓冲并在连接恢复后按序发送的测试
- 增加断线后自动重连、重新发送 `init` 并回放缓冲消息的测试
