# CliGool v1.5.5 - Web Terminal Waiting-CLI Socket Stability

## 🎯 本次更新重点

**CliGool v1.5.5** 重点修复 Web 终端在 CLI 尚未连接或临时断开时反复新建 WebSocket 的问题。现在浏览器会在同一条 relay 连接上保持等待状态与心跳，直到 CLI 返回后自动恢复终端流。

## ✅ 更新内容

### 🌐 无 CLI 时保持 WebSocket 在线
- Web 终端收到 `no_cli` 提示后不再立即触发新的 WebSocket 重连
- 浏览器会保留当前 relay WebSocket，并停留在 waiting-cli 状态
- 在等待期间继续通过现有连接维持心跳

### 🔁 CLI 返回后同连接自动恢复
- 当同一 session 的 CLI 重新连接并发送 `init` 或新的终端输出时
- 浏览器会直接在当前连接上切回 connected 状态
- 同时刷新终端大小同步，避免恢复后尺寸状态不一致

### ❤️ relay 心跳按连接分别跟踪
- relay 改为按 peer 分别记录心跳时间，而不是共享单一时间戳
- CLI 断开后不再错误停止 Web 客户端心跳
- Web-only 等待场景的连接稳定性更高

### 🧪 新增回归测试
- 覆盖“无 CLI 时 Web 连接保持在线，随后收到 init 自动恢复”
- 覆盖“CLI 断开后 Web 连接不关闭，CLI 重连后继续收到 init”
